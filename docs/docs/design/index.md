# 設計

## 開発手法

Yuライブラリは，リファレンスを仕様として位置付け，意味論およびインタフェースを先に確定した上で，その実現のために実装が行われます．

具体的には，あるサブライブラリを開発する際，以下の手順で行われます：

1.  全体の構想を言語化し，意味論をリファレンスに記述して確定する
2.  提供する機能のうち１つのリファレンスを記述してシグネチャや識別子名，振る舞いを確定し，「未実装マーク」を付ける
3.  記述したリファレンスに従ってコードを書き，実装する．
4.  リファレンスに定義された意味論や振る舞いを実装が満たしていれば完成とみなし，「未実装マーク」を取り除く．
5.  2-4の手順をライブラリ完成まで繰り返す

ここで，「未実装マーク」は，仕様が確定しているが実装が未提供であることを示します．以下の通りに表示されます：

!!! warning ""
    この項目は仕様のみ定義されており，現在は未実装です．


## ドキュメントの公開

Yuライブラリの開発では，未実装機能のリファレンスや設計文書を含む，設計過程そのものが公開されます．
これは，思考過程を記録すること，設計判断の透明性を保つことを目的としています．
