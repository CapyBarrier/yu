# 命名規則

ここでは，Yuライブラリにおける命名規則を規定する．
この規則は，名前のケースやスタイルだけでなく，決定自体の基準も規定する．
また，公開される機能だけでなく，ユーザが直接触れることのない実装詳細にも適用される．

## サブライブラリ名

サブライブラリには，そのライブラリが取り扱うものが明瞭に理解できるよう命名する．
厳格に守る必要はないが，意味に適している限りは1つの名詞であることが望ましい．

そのサブライブラリ自体がある対象に特化していて，それが名前となっているような場合を除き，加算名詞は複数形とする．
また，文字数は最大8文字程度とする．

## 識別子名

ユーザ体験と干渉しない限り，短さよりも意味の明瞭さを優先することを基本原則とする．

### 名前空間

ユーザが直接触れてはならないものは，アンダースコア1文字で始まる名前空間に格納される．
例えば，実装詳細は`_detail`名前空間に格納される．

### テンプレートパラメータ

テンプレートパラメータにはアッパーキャメルケース(`UpperCamelCase`)を用いる．

### 非静的メンバ変数

静的でないメンバ変数名には，末尾にアンダースコアを付ける．

### 慣習的な命名

メタ関数のエイリアステンプレート名の末尾に`_t`や`_v`を付けることなど，広く知られている慣習には従う．

### その他

ここに明示されていない識別子には，原則としてスネークケース(`snake_case`)を用いることとする．
