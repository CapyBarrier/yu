# 名前空間

ここでは，Yuライブラリにおける名前空間の設置などの規則を規定する．
サブライブラリと名前空間の関係，名前空間とディレクトリ配置の関係については，ここでは規定しない．

## 公開名前空間

サブライブラリが提供する機能は，そのサブライブラリの名前空間の直下に格納しなければならない．
ユーザが機能領域として独立に理解できる意味単位である場合に限り，下位の公開名前空間を設置することが認められる．

機能そのものを構文的に分類するために名前空間を設置することは，原則として認めない．
例えば，そのライブラリが提供するコンセプトを格納するために`concept`名前空間を置くことは認められない．
ただし，ライブラリが提供するユーザ定義リテラルを分離し格納するための，`literals`インライン名前空間は例外として認める．

## 非公開名前空間

APIに影響しない限り，非公開名前空間の設置や構造に対しては規則を設けない．
ただし，命名については以下の規則に従わなければならない．


非公開名前空間の名前は，先頭がアンダースコア1文字で始まらなければならない．

以下の3つの非公開名前空間は，目的を明確に定めておく．

* `_detail`名前空間 : 実装詳細を格納するために用いる，
* `_unspecified`名前空間 : リファレンスで振る舞いのみが規定される型を格納するために用いる．
* `_exposition`名前空間 : リファレンスで説明専用とした機能の実装を格納するために用いる．

非公開名前空間に対する依存は，互換性を保証しない．
